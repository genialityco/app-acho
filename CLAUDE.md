# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- `npm run start` — start Metro / Expo dev server (use `--clear` to reset cache)
- `npm run android` / `npm run ios` — run native build via `expo run:*` (requires native toolchains; standard Expo Go won't work because of `expo-dev-client` and native plugins like `react-native-pdf` / `react-native-video`)
- `npm run web` — Expo web (Metro bundler)
- `npm run lint` — `expo lint`
- `npm test` — `jest --watchAll` via `jest-expo` preset. Run a single test file with `npx jest path/to/file-test.tsx` or by name with `npx jest -t "<test name>"`
- `eas build --profile <development|preview|production>` and `eas submit --profile production` for store builds (config in `eas.json`)
- OTA updates: `eas update --branch <channel>` — channels are `development`, `preview`, `production`; `runtimeVersion` in `app.json` must match the installed native build for an update to apply

## Architecture

### Routing (Expo Router, file-based)

The `app/` directory drives the navigation tree via `expo-router` v5 with `typedRoutes` enabled.

- `app/_layout.tsx` — root layout. Wraps the entire app in `OrganizationProvider` → `AuthProvider` → `NotificationsProvider` → `PaperProvider` → React Navigation `ThemeProvider`. **Provider order matters:** `AuthContext` consumes `OrganizationContext`, and `NotificationsContext` consumes `AuthContext`.
- `app/login.tsx`, `app/register.tsx` — public routes.
- `app/(app)/_layout.tsx` — auth gate. Redirects to `/login` when `!isLoggedIn`. Also owns app-wide cross-cutting concerns: OTA update check (`expo-updates`), store-version gate (force-update via store URL when installed version is older than the hardcoded latest), Expo push token registration, the global notification modal, and a Firebase Realtime Database listener at `drawer-status-acho` that toggles a remote-controlled `CustomDrawer` (used to push surveys/announcements without shipping a release).
- `app/(app)/(tabs)/_layout.tsx` — bottom tabs: `home` (Novedades), `eventosbefore` (Anteriores), `(index)` (Próximos), `achoinfo`, `menu`. The `home` tab badge shows `unreadCount` from `NotificationsContext`.
- TypeScript path alias `@/*` maps to the repo root (see `tsconfig.json`).

### State / contexts (`context/`)

- `AuthContext` — Firebase Auth (email/password) + a backend "user" record. On Firebase auth state change it calls `searchUsers({ firebaseUid })` and stores both `uid` (Firebase) and `userId` (backend Mongo `_id`). Most API calls key off `userId`, not `uid`. `signUp` creates the Firebase user, then a backend user, then a `Member` linking the user to an organization.
- `OrganizationContext` — currently hardcodes `_id: "66f1d236ee78a23c67fada2a"` (ACHO). Most service calls filter by `organizationId`, so changing this swaps tenants.
- `NotificationsContext` — fetches in-app notifications from the backend and exposes `unreadCount` for the tab badge.

### Backend / services

- `services/api/api.ts` — single Axios instance pointed at `https://lobster-app-uy9hx.ondigitalocean.app`. Comments at the top list local alternatives. There is no auth interceptor: requests are not signed with a Firebase token; the backend identifies the user via `firebaseUid` / `userId` passed in queries or bodies.
- `services/api/*.ts` — one file per domain (events, agenda, attendees, certificates, documents, highlights, members, news, notifications, organizations, posters, speakers, surveys, users). Most expose `searchX(filters)` returning `SearchData<T>` (`{ data: { items, totalItems, totalPages, currentPage } }`).
- `services/firebaseConfig.ts` — Firebase app init for project `global-auth-49737`. Exports `auth` (with `AsyncStorage` RN persistence) and Realtime Database helpers (`db, ref, onValue, set, push, get`). Realtime DB is used only for remote toggles (e.g. `drawer-status-acho`) and ad-hoc logs, not as the primary data store.

### UI

- `react-native-paper` with the custom theme in `theme.tsx` (teal `rgb(0, 105, 115)` primary). The root layout passes this to `PaperProvider`.
- Event detail screens live in `components/event/` (Agenda, Speakers, Posters, Documents, Certificates, Venue, Program). They expect an `Event` shape from `services/api/eventService.ts` whose `eventSections` flags toggle which subsections render.
- Forms use `formik` + `yup`; surveys use `survey-core` / `survey-react-ui` rendered through `components/SurveyForm.tsx`.

### Build / native config

- New Architecture is enabled (`newArchEnabled: true` in `app.json`).
- `plugins/withCustomPodfileConfig.js` is a config plugin (referenced from `app.json` as needed) that mutates the iOS Podfile.
- `metro.config.js` and `babel.config.js` are minimal Expo defaults.
- `android/` is gitignored — it is regenerated by `expo prebuild` / EAS, so don't hand-edit native Android files expecting them to persist.
- Secret JSON files (`google-services.json`, `GoogleService-Info.plist`, `achoapp-*.json`, `global-auth-*-adminsdk-*.json`) are gitignored; they must be present locally for `eas build` / `eas submit` to succeed.

## Conventions worth knowing

- Code comments, log messages, and user-facing strings are in Spanish — match the existing language when adding new ones.
- `firebaseUid` (from Firebase) and `userId` (backend Mongo `_id`) are not interchangeable. Most service calls take `userId`; a few (e.g. `searchUsers`) take `firebaseUid`. Read the service signature before passing IDs.
- The force-update version gate in `app/(app)/_layout.tsx` has a hardcoded Android `latestVersion` (`"1.0.8"` at time of writing) — bumping the Play Store release means updating that string, not just `app.json`.
